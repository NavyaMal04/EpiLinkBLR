# EpiLink BLR 🦟

> Real-time disease outbreak prediction and surveillance system for Bengaluru, built for rapid deployment during a 30-hour hackathon.

EpiLink BLR combines a **TFLite mRDT image classifier** (Phase 1) with a **GCP + Firebase data pipeline** (Phase 2) to detect and predict dengue/malaria outbreaks at the ward level across Bengaluru.

---

## Architecture

```
Field Worker (Flutter App)
        │
        ▼
  mRDT Test Strip Photo
        │
   TFLite Classifier  ──── Phase 1
        │
  Positive / Negative
        │
        ▼
  Firestore (symptom_reports)  ──┐
                                 │── Phase 2
  Civic Hazards (stagnant water) ┤
  Weather Data (Open-Meteo)     ─┘
        │
        ▼
  BigQuery Analytics Engine
        │
        ▼
  Ward-level Risk Scores (0–1)
        │
        ▼
  Dashboard / Alerts
```

## Phase 1 — mRDT Image Classifier

Trains a binary MobileNetV2-based TFLite model to classify malaria/dengue rapid test strip images as **positive** or **negative**.

### Files
| File | Description |
|------|-------------|
| `extract_images.py` | Extracts images from `.docx` files (handles `.wdp` via ImageMagick/Wand) |
| `augment_dataset.py` | Balances dataset via augmentation (flip, rotate, brightness) |
| `train_model.py` | Fine-tunes MobileNetV2 with class weights, exports `.tflite` |
| `convert_to_tflite.py` | Standalone `.tflite` converter with metadata |
| `prepare_flutter_assets.py` | Copies model + labels into Flutter asset structure |
| `mrdt_classifier.dart` | Flutter Dart class for running inference |
| `run_phase1.sh` / `run_phase1.ps1` | End-to-end Phase 1 runner |

### Usage
```bash
# Extract images from docx files
python extract_images.py --positive "CountPositivemRDT.docx" --negative "test strips negative.docx"

# Augment & balance dataset
python augment_dataset.py

# Train model
python train_model.py

# Run all Phase 1 steps
bash run_phase1.sh        # Linux/macOS
.\run_phase1.ps1          # Windows PowerShell
```

### Model Performance
- Architecture: MobileNetV2 (transfer learning, fine-tuned)
- True Positive Rate (Sensitivity): **~88%** (clinically tuned to minimize false negatives)
- Output: `models/mrdt_classifier.tflite` (~2.6 MB)

## Phase 2 — GCP + Firebase Data Pipeline

Provisions the entire cloud backend and populates BigQuery with historical weather, simulated outbreak data, and ward-level risk scores.

### Infrastructure
| Resource | Details |
|----------|---------|
| GCP Project | `epilink-blr` |
| Region | `asia-south1` (Mumbai) |
| BigQuery Dataset | `epilink_blr` |
| Firestore | Native mode, default database |
| Service Account | `epilink-sa` with BigQuery Admin, Firestore, Firebase roles |

### BigQuery Tables
| Table | Rows | Description |
|-------|------|-------------|
| `weather_daily` | 910 | Historical weather per ward (2023) |
| `symptom_reports` | ~5,500 | Synthetic mRDT + symptom reports |
| `civic_hazards` | ~3,700 | Stagnant water / civic hazard reports |
| `ward_risk_scores` | 910 | Daily risk score (0–1) per ward |

### Files
| File | Description |
|------|-------------|
| `setup_gcp.ps1` / `setup_gcp.sh` | Provisions GCP project, APIs, service account, Firestore DB |
| `create_bq_tables.py` | Creates BigQuery schema (4 tables) |
| `fetch_weather.py` | Fetches 2023 weather from Open-Meteo for 10 Bengaluru wards |
| `generate_synthetic_data.py` | Simulates correlated disease reports + outbreak injection |
| `setup_firebase.py` | Initialises Firestore collections |
| `verify_phase2.py` | End-to-end validation of all tables + Firestore |
| `run_phase2.ps1` / `run_phase2.sh` | Sequential Phase 2 orchestrator |

### Usage
```powershell
# Windows (PowerShell) — recommended
.\run_phase2.ps1

# Linux/macOS
bash run_phase2.sh
```

### Prerequisites
- `gcloud` CLI installed and authenticated (`gcloud auth login`)
- Billing account linked in `setup_gcp.ps1`
- Service account key at `./keys/epilink-sa.json` (generated by setup script)
- Python 3.10+ with dependencies: `pip install -r requirements.txt`

## Setup

### 1. Clone the repo
```bash
git clone https://github.com/NavyaMal04/EpiLinkBLR.git
cd EpiLinkBLR
```

### 2. Install dependencies
```bash
pip install -r requirements.txt
```

> **Note:** TensorFlow (Phase 1) requires Python ≤ 3.12. Phase 2 cloud scripts work on Python 3.10+.

### 3. Run Phase 1
```bash
bash run_phase1.sh
```

### 4. Run Phase 2
```powershell
# Edit setup_gcp.ps1 to set your billing account ID, then:
.\run_phase2.ps1
```
